Unit coupling

Dimension unit drives the default weight unit and divisor. The user can override, but the default matches how that region actually ships.

Dimension unitDefault weightDefault mode / divisor
cm / mmkgAir freight, ÷6000 (167 kg/m³)
inlbUS parcel, ÷139

Volume is always reported in m³ regardless of input, since CBM is the global freight standard. Optional carrier rounding rounds each inch up, then the billable weight up to the next lb or 0.5 kg (FedEx and UPS, effective Aug 2025).

Disclaimer levels

Each formula record carries a level (from the corpus schema). The level sets the tone of the note under the readout.

LevelWhenNote pattern
nonePure, undisputed math (CBM, EOQ)No caveat needed.
estimateDepends on a variable constant (DIM divisor)"Divisors vary by carrier, so confirm yours."
regionalValue differs by country (freight class)"US NMFC classes. Check your region."
professional_reviewMoney or compliance (landed cost, duty)"A planning aid. Confirm with your broker or accountant."

SEO and schema slots

  • One H1 per page, unique title and meta description, canonical URL.
  • WebApplication schema on the tool, plus FAQPage and BreadcrumbList.
  • Formula, example, and result rendered in HTML, never behind a JS-only shell.
05   Build recipe

Ship a new calculator in eight steps

Maps to the repo phases: corpus record (1b), formula engine (2), page spec (3), page build (4).

  1. Pick the grounded formula record. It carries inputs, expression, assumptions, a worked example, and a citation. No record, no page.
  2. Lay out inputs. Reuse the field, dimension trio, unit toggle, and select. Match input names to the formula record.
  3. Configure the readout. One primary value in the big amber slot, supporting figures as derived rows, the decisive one marked .charge.
  4. Add Advanced only if needed. Divisors, modes, container fit, actual weight. If the formula is pure, skip it.
  5. Wire copy and units. Reuse the shared format, copy, and unit-coupling utilities. Do not reinvent per tool.
  6. Write the copy blocks. Intro line, formula, worked example from the record, assumptions, and three real FAQs. Set the disclaimer level.
  7. Add metadata and schema. Title, description, canonical, WebApplication, FAQ, Breadcrumb. Link four related bench tiles.
  8. Write deterministic tests. The worked example is a test case. Public result must match the cited example before launch.
